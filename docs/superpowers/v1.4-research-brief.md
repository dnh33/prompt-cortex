## What prompt-cortex Is

A Claude Code plugin that silently injects prompt templates as `additionalContext` via hooks. Pure bash + jq, no runtime dependencies. Designed to complement the superpowers plugin.

- **350 templates** (50 gold, 300 silver) across 6 categories
- **3-phase matching engine** in match.jq (~715 lines of jq)
- **Context awareness** (v1.2): language/framework detection, domain synonyms, complexity scoring, presets
- **NLP preprocessing** (v1.3): 9 conversational patterns, morphological matching, 23 contraction expansions
- **158 tests**, 0 failures

## The Gap Analysis

### The Structural Problem

The scoring formula: `max(action, inferred_action) + max(object, inferred_object) + keyword + signal`

To reach inject (0.70), you need at minimum:
- Action 0.45 + Object 0.25 = 0.70 (direct match both)
- Action 0.30 (inferred) + Object 0.35 + keyword 0.08 = 0.73 (inferred action + direct object)
- Action 0.45 + keyword 0.08 + signal 0.10 + inferred_object 0.20 = impossible (0.45+0.08+0.10 = 0.63, object 0.20 via max = still 0.63)

**The core issue:** prompts without a recognizable object word can never inject. "make it faster" has no object — "it" is a pronoun. The preprocessor infers `optimize` but there's nothing to boost the object score.

### What Needs Solving for v1.4

1. **Object inference from context** — "make it faster" in a Python project → infer object "code" from context. The `$context` variable already has `lang`, `framework`, etc.
2. **Implicit object defaults** — When no object is detected and preprocessor fired, default to "code" (the most common object in coding contexts)
3. **Template affinity boost for preprocessor matches** — When preprocessor infers action=optimize, boost templates with that action beyond the current 0.30
4. **More preprocessor patterns** — "document X", "deploy X", simple imperative forms not covered by P1-P9
5. **The defer gap** — Many prompts land at 0.40-0.55 (defer). Should some defers become injects? Or should the defer behavior change?

## Known Bugs / Technical Debt

1. **`/cx off` not persisted** — cortex-match line 302 hardcodes `cortex_disabled: false`. State not persisted across hook invocations.
2. **P6 filler stripping aggressive** — Strips everything after "I made/wrote", including useful context like "to checkout"
3. **"he's"/"she's" ambiguity** — Always expands to "is", could mean "has"
4. **Contraction expansion is non-word-boundary** — `gsub("it's"; "it is")` would also match inside quoted code
5. **compound_intent telemetry collected but unused** — Data available in output but no command to view it

## Technical Constraints (still apply)

- Pure bash + jq — no new runtime dependencies
- jq ONIG regex engine — no `\b` word boundaries
- Hook must complete in <2 seconds
- `$state` is string (--arg), not object
- Windows + macOS + Linux compatibility
- No LLM calls in Layer 1 (the matching engine)

## Architecture Reference

```
hooks/cortex-match (bash)
  → reads stdin JSON from Claude Code
  → calls jq -f scripts/match.jq with 8 args
  → match.jq pipeline:
      0. Escape check (--raw, /cx off)
      1. Leave-it-alone (13 detectors)
      Preprocessor (9 patterns, contraction expansion)
      2. keyword_candidates_enhanced (inverted index + cleaned_terms + morph_map)
      3. score_candidate (action/object/keyword/signal/inferred/penalties)
      4. context_filter (lang/framework/boost/suppress/affinity/branch)
      5. Threshold decision (inject ≥ 0.70, defer ≥ 0.40, else skip)
  → reads template body, enforces token budget
  → outputs hook JSON with additionalContext
```

## Council Questions

These are the questions the brainstorm should address:

1. **How do we close the inference-to-injection gap without increasing false positives?**
2. **Should v1.4 focus on NLP depth (more patterns, context-aware object inference) or breadth (adaptive confidence, composition, marketplace)?**
3. **Is the 0.70 inject threshold still right? Should we have a "soft inject" tier between defer and inject?**
4. **What's the highest-impact single feature for user experience?**
5. **Should we break the "no LLM in Layer 1" constraint for specific, bounded cases?**
6. **What can we learn from compound_intent telemetry data for template composition design?**
7. **What's the right scope for v1.4 — how many features, how many sessions?**
