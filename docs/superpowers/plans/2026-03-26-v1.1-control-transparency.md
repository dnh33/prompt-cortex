### Task 2: Intent Signal Fix

The YAML parser in build-index.sh double-escapes backslashes: `\s` in template frontmatter becomes `\\s` in index.json, breaking regex intent signals.

**Root cause:** Line 54 in build-index.sh: `val="${val//\\/\\\\}"` escapes ALL backslashes. But template frontmatter already has literal `\s`, `\b` etc. that should pass through as single-escaped for jq regex.

**Fix:** Only escape backslashes that are NOT already part of regex escape sequences (`\s`, `\d`, `\w`, `\b`, `\n`, `\t`, `\(`, `\)`, `\{`, `\}`, `\[`, `\]`, `\+`, `\*`, `\?`, `\.`, `\^`, `\$`, `\|`).

**Files:**
- Modify: `scripts/build-index.sh:46-56` (array item escape logic) and `scripts/build-index.sh:88-95` (scalar value escape logic)
- Test: `tests/run-tests.sh` (new test)

- [ ] **Step 3: Diagnose the exact double-escape**

First, confirm what's actually in the index:

```bash
# Show raw bytes of a signal in index.json
jq -r '.templates[] | select(.id == "coding-001") | .intent_signals[0]' data/index.json | cat -v
# Also show the raw JSON representation

```

**Expected finding:** The JSON string contains `\\\\s` (four-char escape = two backslashes). When jq reads this, `test()` receives `\\s` (literal backslash+s) instead of `\s` (regex whitespace).

**Why it happens:** The bash YAML parser (line 54) does `val="${val//\\/\\\\}"` which turns `\s` → `\\s`. Then this is embedded in a JSON string as `"\\\\s"` (JSON-escaping the two backslashes). When jq loads the index, it JSON-unescapes to `\\s` — one level too many. The regex engine gets a literal backslash followed by 's', not the `\s` whitespace class.

- [ ] **Step 4: Fix — remove backslash escaping from YAML parser**

The fix: remove the backslash escape line from BOTH the array item handler and the scalar value handler. The bash `printf '%s'` that builds the JSON string will embed `\s` directly, which is valid JSON (unrecognized escape sequences are passed through by most parsers) and jq will read it as the regex pattern `\s`.

**Array item section (build-index.sh around line 54)** — change:
```bash
      # Escape backslashes and quotes for JSON
      val="${val//\\/\\\\}"
      val="${val//\"/\\\"}"
```
to:
```bash
      # Escape quotes for JSON embedding
      # NOTE: Do NOT escape backslashes — regex patterns like \s, \b need
      # to pass through as-is for jq's test() function. The raw \s in the
      # bash string becomes \s in the JSON, which jq reads as regex \s.
      val="${val//\"/\\\"}"
```

**Scalar value section (build-index.sh around line 95)** — same change:
```bash
      # Escape for JSON
      val="${val//\\/\\\\}"
      val="${val//\"/\\\"}"
```
to:
```bash
      # Escape quotes for JSON embedding (see array section for backslash rationale)
      val="${val//\"/\\\"}"
```

**Risk assessment:** No template fields currently contain literal backslashes that aren't regex patterns. If a future template has a Windows-style path like `C:\Users`, it would need manual escaping in the frontmatter. This is acceptable — regex patterns are the 99% case for backslashes in template frontmatter.

Add synonym lookup helpers at the top of match.jq (after `def parsed_state:`):

```jq
# Synonym expansion — checks if a word is a synonym for a canonical action/object
# $intents is passed via --slurpfile (array with one element) or --argjson intents "null"
# CRITICAL: jq crashes if $intents is not declared at all. cortex-match MUST always
# pass either --slurpfile intents <file> or --argjson intents "null".
def intents_data:
  ($intents // null) | if . == null then null elif type == "array" then .[0] else . end;

def is_action_synonym($word; $canonical):
  intents_data as $i |
  if $i == null then false
  else
    ($i.action_synonyms[$canonical] // []) |
    map(ascii_downcase) |
    any(. == $word or ($word | startswith(.)) or (. | startswith($word)))
  end;

def is_object_synonym($word; $canonical):
  intents_data as $i |
  if $i == null then false
  else
    ($i.object_synonyms[$canonical] // []) |
    map(ascii_downcase) |
    any(. == $word or ($word | startswith(.)) or (. | startswith($word)))
  end;
```

    if [[ "$body_word_count" -gt "$MAX_WORDS" ]]; then
      # Truncate at sentence boundary before MAX_WORDS

      word_count=0
      last_sentence_end=""
      while IFS= read -r line; do
        line_words=$(printf '%s' "$line" | wc -w | tr -d ' ')
        word_count=$((word_count + line_words))
        if [[ -n "$truncated" ]]; then
          truncated+=$'\n'"$line"
        else
          truncated="$line"
        fi
        # Track last sentence boundary (line ending with . ! or ?)
        if [[ "$line" =~ [.!?]$ ]]; then
          last_sentence_end="$truncated"
        fi
        if [[ "$word_count" -ge "$MAX_WORDS" ]]; then
          break
        fi
      done <<< "$template_body"

      # Use last sentence boundary if available, otherwise hard truncate
      if [[ -n "$last_sentence_end" ]]; then
        template_body="$last_sentence_end"
      else
        template_body="$truncated"
      fi

      # Log truncation
      log_truncation="true"
