# prompt-cortex — Development Guide

## What This Is

A Claude Code plugin that silently injects prompt templates as `additionalContext` via hooks. Pure bash + jq, no runtime dependencies. Designed to complement the superpowers plugin.

## Project Structure

```
prompt-cortex/
  .claude-plugin/
    plugin.json                     # Plugin manifest (name, version, author)
  hooks/
    hooks.json                      # Hook registrations (SessionStart + UserPromptSubmit)
    run-hook.cmd                    # Cross-platform polyglot wrapper (bash + batch)
    cortex-match                    # UserPromptSubmit hook — main matching + injection
    cortex-session-init             # SessionStart hook — index validation + project detection
  data/
    intents.json                    # Action/object taxonomy with synonym maps
    index.json                      # GENERATED — pre-built inverted index (do not edit manually)
    prompts/
      coding/                       # coding-001 to coding-150 (78 files)
      ai-workflows/                 # ai-051 to ai-150 (72 files)
      research/                     # research-001 to research-050
      automation/                   # automation-001 to automation-050
      content/                      # content-001 to content-050
      productivity/                 # productivity-001 to productivity-050
  scripts/
    match.jq                        # Core matching engine — leave-it-alone + 3-phase scoring
    build-index.sh                  # Index compiler — YAML parser + inverted index builder
    validate-template.sh            # Template frontmatter validator
  tests/
    run-tests.sh                    # Integration test suite (33 tests)
  commands/
    debug.md                        # /cortex:debug — show last match result
    list.md                         # /cortex:list — list templates by category
    sync.md                         # /cortex:sync — rebuild index
  docs/
    2026-03-25-prompt-cortex-design.md          # Original design spec
    how-the-index-works.md                      # Inverted index architecture deep dive
    superpowers-skills-reference.md             # Superpowers skill mapping guide
    roadmap.md                                  # Future features (v1.1, v1.2, v2.0)
    prompts.txt                                 # Original 300 prompts source material
    superpowers/plans/                          # Implementation plans and reference docs
```

## Key Files You Must Understand

- **`scripts/match.jq`** — The brain. Leave-it-alone detector (12 signals), 3-phase matching, weighted scoring. Read `docs/how-the-index-works.md` for the full explanation.
- **`hooks/cortex-match`** — The spine. Reads stdin, calls match.jq, reads template body, outputs hook JSON, manages state and telemetry.
- **`scripts/build-index.sh`** — The compiler. Pure-bash YAML parser → JSON → inverted index. Uses temp files to avoid arg length limits.

## Do NOT Edit Manually

- **`data/index.json`** — Generated by `build-index.sh`. Run `bash scripts/build-index.sh` to regenerate.
- **Template IDs** — Must match filename. `coding-001.md` → `id: coding-001`.

## Technical Constraints

- **jq regex**: `\b` word boundaries do NOT work in jq's ONIG engine. All patterns MUST use `(^|[^a-zA-Z])word([^a-zA-Z]|$)`.
- **printf over heredoc**: Bash 5.3+ has a heredoc variable expansion bug. Use `printf` for all JSON output.
- **Extensionless scripts**: Hook scripts are `cortex-match` not `cortex-match.sh` — prevents Windows auto-detection interference.
- **State as string**: `$state` is passed via `--arg` (always a string in jq). match.jq uses `fromjson` to parse it.
- **Temp files for large data**: build-index.sh writes to temp files and uses `--slurpfile` to avoid "Argument list too long" with 350+ templates.

## Template Schema

### Required Fields
`id`, `name`, `category`, `intent`, `action`, `object`, `triggers` (3+ entries)

### Recommended Fields
`intent_signals`, `negative_signals`, `quality_tier` (gold/silver/bronze), `token_overhead`, `min_confidence`, `composable_with`, `composition_role`, `compatible_with` (superpowers skills), `conflicts_with`

### Valid Values
- **Actions**: create, review, debug, refactor, explain, test, document, optimize, design, fix
- **Objects**: code, function, file, component, test, PR, commit, API, schema, prompt, config, error, architecture, database
- **Categories**: coding, ai-workflows, research, automation, content, productivity

## Quality Tiers

- **Gold** (coding-001 to coding-028, ai-051 to ai-072): Our purpose-built templates with full frontmatter, tuned triggers, intent signals, superpowers mapping
- **Silver** (coding-101+, ai-101+, research-*, automation-*, content-*, productivity-*): Adapted from external prompt library, solid but less tuned

## Development Commands

```bash
# Run tests
bash tests/run-tests.sh

# Validate all templates
bash scripts/validate-template.sh data/prompts/**/*.md

# Rebuild index
bash scripts/build-index.sh

# Test matching
jq -f scripts/match.jq --arg prompt "review my code" --arg state "null" --arg cwd "" data/index.json

# Test hook end-to-end
echo '{"session_id":"test","prompt":"review my code","cwd":"/tmp"}' | CLAUDE_PLUGIN_ROOT="$(pwd)" bash hooks/cortex-match
```

## Superpowers Integration

Every template has a `compatible_with` field mapping to superpowers skills. See `docs/superpowers-skills-reference.md` for the full mapping guide and coverage matrix. When adding templates, always map to relevant superpowers skills.

## Known Issues

- **Intent signals double-escaped**: build-index.sh YAML parser over-escapes `\\s` to `\\\\s`. Matching works via action/keyword scoring but regex signals are broken. Fix tracked in `docs/roadmap.md`.
- **"ok" returns "skip" not "suppress"**: Continuation detector scores 0.45, below 0.60 threshold. Functionally equivalent (nothing injected).
