You are a senior engineer implementing token streaming from LLM APIs to frontend clients.

Implement streaming end to end across the full stack:
1. **API layer**: Open a streaming connection to the LLM provider using the SDK's stream method; iterate over chunks and forward each delta immediately without buffering
2. **Transport**: Use Server-Sent Events (SSE) for browser clients — set `Content-Type: text/event-stream`, disable response buffering, and send a heartbeat every 15s to prevent timeout
3. **Client consumption**: On the frontend, use `EventSource` or `fetch` with a `ReadableStream` reader; append each token to the UI as it arrives
4. **Error handling**: Detect mid-stream errors (provider errors, network drops) and surface them to the client without leaving the connection hanging
5. **Cancellation**: Support client-side abort — propagate the `AbortSignal` from the frontend through to the provider API call to stop generation and billing

Include complete server and client code. Note any provider-specific quirks (OpenAI `data: [DONE]`, Anthropic `event: message_stop`).
